# 지식그래프의 핵심 개념

## 지식그래프란?
node와 edge로 구성.<br>
▪️ node 에는 개념과 사물의 정보를 담고 <br>
▪️ edge 에는 관계의 의미를 담음 <br>

## 핵심 요소
### ⚫️ 엔티티 (Entity)
엔티티는 지식그래프의 노드에 해당함. 현실세계에서 식별 가능한 개체를 의미함. <br>

```
[세종대왕]    ← 역사적 인물 (엔티티)
[한글]        ← 문자 체계 (엔티티)
[조선]        ← 국가/왕조 (엔티티)
[1443년]      ← 시점 (엔티티)
```
#### ▪️ 엔티티의 특징 <br>
- 고유한 식별자를 가짐<br>
- 여러 속성을 가질 수 잇음 <br>
- 다른 엔티티와 관계를 가짐 <br>

### ⚫️ 관계 (Relationship)
**관계**는 두 엔티티 사이의 연결을 의미있게 정의하는 것.
#### ▪️ 관계의 특징
- 명확한 의미를 가진 이름
- 방향이 있을 수 있음 (a->b != b->a)
- 자체적인 속성을 가질 수 있음

### ⚫️ 속성 (Property / Attribute)
엔티티와 관계 모두 속성을 가질 수 있음.<br>
#### 속성의 예시
```
엔티티 속성:

[세종대왕]
├── 본명: 이도
├── 생년: 1397년
├── 몰년: 1450년
├── 재위기간: 1418-1450
└── 묘호: 세종


관계 속성:

[세종대왕] ──창제── [한글]
              │
              ├── 창제년도: 1443년
              ├── 반포년도: 1446년
              └── 초기이름: 훈민정음
```

### ⚫️ 트리플(Triple) 구조
지식 그래프의 가장 작은 단위는 트리플.<br>
```
┌-----------┐      ┌-----------┐     ┌-----------┐ 
|    주어    |      |    관계    |     |    목적어   |
| (subject) | -->  |(Predicate)| --> |  (Object) |
└-----------┘      └-----------┘     └-----------┘

트리플	주어	관계	목적어
1	세종대왕	창제	한글
2	세종대왕	아버지	태종
3	한글	글자수	28자
4	서울	수도	대한민국
```
#### 트리플 지식 표현
복잡한 지식도 트리플조합으로 표현 가능.<br>
```
"세종대왕은 조선의 4대 왕으로, 한글을 창제했다"

→ 트리플로 분해:
  (세종대왕, 왕조, 조선)
  (세종대왕, 대수, 4대)
  (세종대왕, 창제, 한글)

```
## ⚫️ 스키마 (schema)
**스키마**는 지식 그래프의 **설계도** 임.  어떤 종류의 엔티티와 관계가 있는지 정의함.

## ⚫️ 다단게 추론
지식그래프에서는 경로(path)를 따라 정보를 찾음.

### ▪️ 1홉 (one-hop) VS 다중홉 (Multi-hop)
```
1홉 질문:
"세종대왕이 창제한 것은?"
-> [세종대왕] -창제-> [한글]

= 1단계 탐색으로 해결

다중홉 질문: 
"한글을 창제한 왕의 아버지가 세운 기관은?"
-> [한글] <-창제- [세종대왕] <- 아들 - [태종] -설립-> [의금부]

= 3단계 탐색 필요
```
#### 지식 그래프가 필요한 순간
- 모호한 질문 해결<br>
- 불완전한 질문의 이해<br>
- 비교/대조 질문 <br>
- 역방향 질문 <br>
